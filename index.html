동물병원 ERP 시스템(atti) 2024-04-29 ~ 2024-06-04

프로젝트 개요 개발 내용
제목 : 동물병원 ERP 시스템(atti)
기간 : 2024-04-29 ~ 2024-06-04
인원 : 4명
 - 모델1 방식을 사용하여 ERP 시스템 구현
- 페이지 내에서 관리자, 의사(간호사), 퇴사자 기능별 차등 부여
- 퇴사자는 삭제 조치되지 않지만 로그인 등 어떠한 기능도 사용할 수 없음
담당 기능
  1. 보호자 / 펫 조회
    • 보호자 / 펫의 정보 조회
      - 펫 이름, 보호자 이름, 연락처 뒷자리, 등록일(최초), 접수한 시간을 출력합니다.
    • 보호자 / 펫의 정보 검색
      - 펫 이름 / 보호자 이름 / 연락처 등을 검색할 수 있습니다.
  2. 보호자 조회
    • 보호자 정보 조회
      - 보호자의 이름, 연락처 뒷자리, 보호자 명의로 등록되어 있는 펫의 수, 최초 방문 등록일을 출력합니다.
    • 보호자 정보 검색
      - 보호자의 이름 / 연락처 등을 검색할 수 있습니다.
  3. 펫 조회
    • 펫 정보 조회
      - 펫 이름, 보호자 이름, 연락처 뒷자리를 출력합니다.
    • 보호자 정보 검색
      - 보호자의 이름 / 연락처 등을 검색할 수 있습니다.
  4. 보호자 등록 및 수정
    • 보호자 등록
      - 검색 조회 시 정보가 없다면 이름 / 연락처 / 주소의 정보를 입력하여 보호자를 등록할 수 있습니다.
      - 보호자 등록 성공 시 보호자 상세 페이지로 이동합니다.
    • 보호자 정보 수정
      - 보호자의 연락처 / 주소를 수정할 수 있습니다.
  5. 보호자 상세 페이지
    • 보호자 상세 페이지
      - 고객 번호, 보호자 이름, 연락처, 주소를 출력합니다.
    • 보호자의 펫 리스트
      - 보호자와 펫 테이블을 JOIN하여 보호자 상세 정보 하단에 명의로 등록되어 있는 펫 리스트 출력합니다.
      - 펫의 이름, 최근 진료일, 접수하기 버튼을 출력합니다.
      - 초진 등의 사유로 진료 이력이 없을 경우 "진료 이력 없음" 문구가 출력됩니다.
  6. 펫 등록 및 수정
    • 펫 등록
      - 보호자 상세 페이지에서 펫 등록이 가능합니다.
      - 포유류 / 파충류 / 조류의 분류를 선택
      - 강아지 / 고양이 등의 종류를 직접 입력
      - 펫의 이름을 직접 입력
      - 펫의 나이에 따른 진료를 위해 생년월일을 입력
    • 펫 정보 수정
      - 펫의 정보 중 이름만 수정이 가능합니다.
  7. 펫 상세 페이지
    • 펫 상세 페이지
      - 펫 번호, 펫 분류, 펫 종류, 펫 이름, 생년월일, 보호자의 이름, 보호자 연락처를 출력합니다.
      - 수정하기 버튼 클릭 시 펫 수정 페이지로 이동합니다.
  8. 진료 상세 페이지
    • 고객 정보
      - 보호자 이름, 펫의 분류, 펫의 종류, 펫 이름 출력합니다.
      - 접수 번호에 따른 접수 내용과 접수된 시간을 출력합니다.
    • 진료 내용 등록
      - JavaScript를 이용하여 현재 시간을 입력창에 출력합니다.
      - 진료 내용을 입력 후 저장하면 진료 이력에 출력됩니다.
    • 진료 이력
      - 해당 진료가 아닌 과거 진료 이력을 조회하여 출력합니다.
      - 내용을 등록한 시간과 등록한 진료 내용이 출력됩니다.
    • 수술 내용 등록
      - 수술(안과, 내과, 외과)를 선택 / 수술 내용을 입력 / 수술 예정 시간을 선택하여 입력합니다.
      - 이미 등록된 수술이 있다면 해당 기능은 비활성화됩니다.
    • 수술 내용 조회
      - 등록된 수술 내용이 있다면 조회 기능만 활성화됩니다.
    • 수술 내용 수정
      - 수술 종류와 수술 내용을 수정합니다.

제목 : 쇼핑몰
기간 : 2024-04-00 ~ 2024-04-00

- 로그인, 로그아웃 기능 구현
- 고객, 직원, 관리자의 기능 분리
- 회원 가입 및 회원 정보 수정, 회원 탈퇴 구현
- 10페이지 단위의 페이징 구현
담당 기능
  1. 직원
    • Session
      - 세션 기반의 로그인 / 로그아웃 기능을 구현하였습니다.
      - 세션 만료 시 로그인 페이지로 이동합니다.
    • 회원 가입
      - 로그인 페이지에서만 회원 가입 메뉴가 출력됩니다.
      - 중복된 아이디로는 가입할 수 없습니다.
      - ID, PW, 이름, 담당 업무, 고용일 등을 입력합니다.
    • 직원 리스트
      - ID, 이름, 담당 업무, 고용일을 출력합니다.
      - 이름 클릭 시 상세 페이지로 이동합니다.
    • 직원 상세 페이지
      - 한 직원의 ID, 등급, 이름, 담당 업무, 고용일을 출력합니다.
    • 카테고리
      - 제품의 카테고리를 등록합니다.
      - 기등록된 카테고리를 조회합니다.
      - 기등록된 카테고리를 삭제합니다.
    • 제품 등록
      - 카테고리, 제품명, 이미지, 수량, 정보를 등록합니다.
    • 제품 리스트
      - 전체 제품 목록을 조회합니다.
    • 제품 상세 페이지
      - 제품의 카테고리를 등록합니다.
      - 기등록된 고객의 댓글 중 문제되는 댓글을 삭제합니다.
    • 고객 목록
      - 전체 고객 목록을 조회합니다.
    • 주문
      - 고객이 주문한 제품의 목록을 조회합니다.
      - 주문 번호, 제품명, 주문 수량, 제품 가격, 주소, 주문 일자, 총 가격, 배송 상태를 출력합니다.
      - '주문 완료' 상태인 제품을 '배송 중'으로 변경합니다.
  2. 관리자
    • 직원 리스트
      - 직원 리스트 페이지에서 직원의 계정을 ON/OFF로 활성화/비활성화할 수 있습니다.
  3. 고객
    • Session
      - 세션 기반의 로그인 / 로그아웃 기능을 구현하였습니다.
      - 세션 만료 시 로그인 페이지로 이동합니다.
    • 회원 가입
      - 로그인 페이지에서만 회원 가입 메뉴가 출력됩니다.
      - 중복된 아이디로는 가입할 수 없습니다.
      - E-Mail, PW, 생년월일, 성별 등을 입력합니다.
    • 고객 상세 페이지
      - E-Mail, 이름, 생년월일, 성별이 출력됩니다.
      - PW를 수정할 수 있습니다.
      - PW를 확인 후 회원 탈퇴를 진행합니다.
    • 제품 리스트
      - 쇼핑몰에 등록되어 있는 제품의 리스트를 조회합니다.
    • 제품 상세 페이지
      - 제품 번호, 카테고리, 제품명, 제품 설명, 제품 가격, 제품 수량, 제품 등록일이 출력됩니다.
      - 해당 제품을 구매한 고객은 리뷰를 작성할 수 있습니다.
      - 리뷰를 작성한 고객은 본인의 리뷰를 삭제할 수 있습니다.
    • 주문 리스트
      - 고객이 주문한 제품 리스트를 출력합니다.
      - 배송이 완료되면 고객이 '배송 완료'로 상태를 변경합니다.
